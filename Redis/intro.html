# Redis 数据类型

  1. Redis属于NoSql数据库高并发 内存型
  2. Redis支持五种数据类型：
    • string（字符串）
    • hash（哈希）
    • list（列表）
    • set（集合）
    • zset(sorted set：有序集合)。

# 特点

  • 存储结构 - 字典类型(类似于对象数组) 但是键名(key)可以是其他数据类型

  • 内存存储与持久化

  • 功能丰富(缓存 队列系统)

# 常用命令

redis.conf redis的配置文件

  1. 字符串 redis-server 启服务redis redis-cli 命令行操作数据库 select [index] 可以切换数据库 keys * 获取所有key值
    exists key判断是否存在 del key删除 type key 获取数据类型 incr key 加1 incrby key num 增加指定数值 incrbyfloat key num 增加小数 decr key 减1 decrby key num 减指定数值 decrbyfloat key num 减小小数 append key 字符串拼接 strlen key 获取key的字符串长度

  2. 散列(hash) hset obj key value 返回1 设置成功 hgetall obj 获取所有字段 hexists obj key 判断 obj中的key存在吗 hincrby obj key value 增加多少 hdel obj key 删除key

  3. 列表类型(双向链表) lpush key value rpush key value 左右添加 llen key 获取长度 lrange key start stop 范围获取 lpop key 删除左边第一个 lrem key count value 批量删除count个value值 lindex key index 获取指定索引的值 lset key index value 修改指定位置的值为value ltrim key start end 保留strat-end 之间的值 其他的删除

  4. 集合类型(set) 没有顺序 每个元素都是唯一的 sadd key member [...member] srem key member 增加删除元素 smember key 查看集合 sismember key member 判断member是否存在key中 返回1(存在) 0(不存在) sdiff key1 keys差值 sinter key1 key2 交集 sunion key1 key2 并集 sdiffstore dest key key1 差值并存储
    sinterstore dest key key1 交集并存储
    sunionstore dest key key1 并集并存储
    srandmember key count 随机获取count个key中的值 count 负值 可以重复 spop key count 随机弹出

  5. 有序集合 zadd children index1 value1 index2 value2 添加有序集合 zcard children 获取有序集合的长度 zrange children min max [withscores]

  6. 事务(multi) watch

  • 一个事务中的命令要么都执行，要么都不执行
  • WATCH命令可以监控一个或多个键，一旦其中有一个键被修改（或删除），之后的事务就不会执行 (避免竞态)
  1. 缓存
    expire key time 设置过期时间
    ttl key 查询还剩多久过期 返回 -1 永久生效 -2 key 不存在 presist key 取消过期时间永久生效(重新set 也会取消过期时间)

# 集合类型

sadd key value1 (value1 value1 value1) // 添加集合

SCARD key  // 获取集合的成员数 length

smembers key // 返回集合中的所有成员

SDIFFSTORE destination key1 [key2]  // 返回给定所有集合的差集并存储在 destination 中

srandmember key [count] count 正负表示能不能重复

spop key 随机删除一个元素
