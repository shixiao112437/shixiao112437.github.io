# css优化

# 将css提取到独立的文件中

# 使用说明

  1. mini-css-extract-plugin 用于css'提取为独立的文件

  2. npm i mini-css-extract-plugin -D

  3. 引入

     ```js
 const miniCssExtractPlugin = require('mini-css-extract-plugin')
 ```

  4. plugins:[
    new miniCssExtractPlugin({
      filename: '[name].css'  // 入口文件的名字
    })
]

# 自动添加css前缀

# 使用说明

  1. 使用postcss ,需要用到postcss-loader autoprefixer 插件
  2. 安装 npm i -D postcss-loader autoprefixer
  3. 修改配置文件
    
module:{
   rules:[
       {
           test:/\.css$/,
           use:['style-loader','css-loader','postcss-loader']
       }
   ] 
}
  4. postcss 配置说明(.postsrc,或者post css.config.js)
    module.exports = {
        plugins:[
            require('autoprefixer')
        ]
    }

# 压缩css

需要使用optimize-css-assets-webpack-plugin插件来完成css压缩

但是由于配置css压缩时会覆盖掉webpack默认的优化配置，导致JS代码无法压缩，所以还需要手动把JS代码压缩插件导入进来：terser-webpack-plugin

  1. 安装 npm i -D optimize-css-assets-webpack-plugin terser-webpack-plugin
  2. 导入插件
const TerserJSPlugin = require('terser-webpack-plugin')
const OptimizeCSSAssetsPlugin = require('optimize-css-assets-webpack-plugin')
  1. 在webpack配置文件中添加配置节点
optimization: {
  minimizer: [new TerserJSPlugin({}), new OptimizeCSSAssetsPlugin({})],
},

# js优化

Code Splitting 可以将代码分离得到不同的bundle中,然后按需加载

  1. 手动配置多入口配置
    module.exports = {
        // entry:"./src/index.js", // 入口文件
        entry:{
            main:"./src/main.js",
            main:"./src/other.js",
        }, // 多入口文件打包
        output:{
            path:path.join(__dirname,'./dist') , // 绝对路径
            filename:'bondle.js' // 打包文件名
        },
        mode:"development",// "production" 
    }
  1. 抽离公共代码 当两个模块引用相同的模块的时候,他们在两个模块都有

在webpack.base.js中

module.exports = {
    optimization:{
        splitChunks:[
            chuns:"all", // 所有的引用都分割
        ]
    },
    entry:{
        ...
    }
}
