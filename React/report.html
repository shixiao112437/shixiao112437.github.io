# setState

  1. 不可变值(不能直接修改this.state中的值)
  2. 可能是异步更新
    • 在定时器中是同步的(在componentWillUnmount中销毁定时器)
    • 在自定义的dom事件中是同步的(在dom上绑定的事件回调中使用,在componentWillUnmount中注销事件)
    • 直接使用是异步的(在setState的第二个参数中可以拿到最新的值)
  3. 可能会被合并
    • 传入对象会被合并 只会执行一此
    • 传入函数不会被合并,

# shouldComponentUpdate()

  1. 默认返回true 在React默认情况下 父组件更新 子组件则无条件更新
  2. 不可变值,当你直接修改state中的值时 , 在SCU中的this.state.key 会和 nextState中的key 一样 可能会发生bug
   shouldComponentUpdate(nextProps,nextState){
       if(nextState.name!==this.state.name){
           return true  // 可以渲染
       }
       return false  // 不重复渲染

   }
